# 复杂 SVG 规范模板生成计划

## 1. 任务摘要
创建一个较复杂的 SVG 示例，包含连接线、带阴影的方块、带格式的文本以及渐变等元素。该 SVG 将严格遵循 `ppt-master` 项目的约束规范，可作为其他程序生成合法 SVG 的参考模板。最后将该 SVG 转换为 PPTX 供用户下载验证。

## 2. 需求与规范分析
根据 `shared-standards.md` 的规范，该复杂示例需要展示以下关键特性：
1. **画布与背景**：严格的 1280x720 尺寸，使用 `<rect>` 绘制底层背景，不使用 `rgba` 而是 `fill-opacity`。
2. **元素分组**：必须使用带有明确 `id` 的 `<g>` 标签对逻辑模块（如卡片、标题）进行分组（如 `<g id="card-1">`），以便 PPT 正确识别动画锚点和图层。
3. **文本排版**：单行文本必须使用单一 `<text>` 标签，局部强调（颜色/加粗）必须嵌套 `<tspan>` 且不带换行坐标。
4. **连接线与箭头**：在 `<defs>` 中定义符合白名单形状（如三角形 `M0,0 L10,5 L0,10 Z`）的 `<marker>`，通过 `marker-end` 绑定到 `<line>` 或 `<path>` 上，并确保 `fill` 与 `stroke` 颜色一致。同时展示虚线（`stroke-dasharray`）。
5. **阴影效果**：使用标准的 `<filter>` 配合 `feGaussianBlur` 和 `feOffset` 实现卡片的柔和阴影，避免使用被禁用的 `<mask>`。
6. **渐变效果**：使用 `<linearGradient>` 填充强调色块或按钮。

## 3. 具体实施步骤
1. **初始化项目**：使用 `project_manager.py init complex_svg_demo --format ppt169` 初始化工作区。
2. **编写复杂的 SVG 文件**：将设计好的 SVG 写入 `projects/complex_svg_demo_ppt169_<date>/svg_output/page_01.svg`。
    - **内容设计**：设计一个“三步流程图”或“架构图”。包含顶部标题区、3个带有投影和渐变的圆角矩形卡片、带有 `<tspan>` 强调的数据指标文本、以及使用 `marker-end` 连接卡片的实线与虚线箭头。
3. **编写备注文件**：创建基础的 `notes/total.md` 满足导出流程要求。
4. **执行质量检查**：运行 `svg_quality_checker.py` 确保我们编写的复杂 SVG 符合规范（0 Errors）。
5. **执行导出流水线**：依次运行 `total_md_split.py`、`finalize_svg.py` 和 `svg_to_pptx.py`。
6. **返回结果**：向用户提供生成成功的 `.pptx` 文件的下载链接。

## 4. 假设与决策
- **图形设计**：为了最大限度覆盖规范，设计将包含：1个全局背景，1个标题组，3个节点组（带标准阴影），2根连接线（1根实线曲线，1根虚线直线），1个渐变按钮/标签。
- **字体后备**：文本将包含安全的 fallback 字体栈（如 `Microsoft YaHei, Arial`）。

## 5. 验证标准
- 转换脚本执行成功，不抛出任何有关 SVG 解析的 Error 或 Warning。
- 最终生成的 PPTX 中：阴影呈现为原生形状阴影、文本为单行多样式原生文本框、连接线依然保持为原生可编辑连接符。
