# Request
请绘制一份 PPT Master 项目的完整生成流程图（Flowchart / Architecture Diagram）。
该图需要包含以下核心层级和执行链路，展示该项目是如何把源文档最终变为原生可编辑 PPTX 的：

1. **Phase A: 内容理解与设计规划（Strategist 阶段）**
   - 源内容转换 (PDF/Word/URL -> Markdown)
   - 创建项目结构
   - 模板处理（如果用户指定）
   - Strategist 八项确认（Canvas, Audience, Style, Color, Typography 等），这是一个带有阻塞性质的交互确认（Confirm UI / Chat）
   - 输出设计契约：`design_spec.md`（人类可读叙述）与 `spec_lock.md`（机器可读防漂移约束）

2. **Phase B: 视觉执行与资源获取（Executor 阶段）**
   - 图像获取 (Image Acquisition): 如果包含 ai 或 web 图片，调用 image_gen.py / image_search.py 
   - 启动 Live Preview 本地实时预览
   - 批量预读模版 (Pre-generation Batch Read)
   - 逐页生成 SVG：**LLM 强制一页接一页串行生成 SVG**，每生成一页前必须重新读取 `spec_lock.md`，输出到 `svg_output/`
   - 质量校验门 (Quality Check Gate): `svg_quality_checker.py`，必须 0 error 才能进入下一步
   - 生成讲稿笔记 (Speaker Notes) -> `notes/total.md`

3. **Phase C: 后处理与导出（Post-processing & Export 阶段）**
   - 分拆讲稿 (Split Notes)
   - SVG 后处理 (Finalize SVG): 把图标内联、文字压平、计算原生圆角等，产出 `svg_final/`
   - 导出为原生 PPTX (svg_to_pptx): 元素级地把 SVG 翻译为 DrawingML，产出 `presentation.pptx`

要求：
- 显式表达串行的流水线（Pipeline）结构。
- 凸显出 `spec_lock.md` 在 Executor 阶段循环中的“防漂移、每页必读”的作用。
- 强调后处理与导出的三步（Split -> Finalize -> Export）是脚本执行而非 LLM 执行。
