# Request
请绘制 PPT Master 中 Executor（执行者）在生成单页幻灯片时的“三重滤镜（约束）”执行逻辑架构图。

核心逻辑结构与依赖链路：
1. 第一层滤镜（全局设定）：Mode & Visual Style（模式与视觉风格）
   - 角色：提供全局建筑规范与外立面材质（如极简风、禁止投影）。
   - 属性：全局必有，贯穿所有页面。
   - 下游关系：约束后续所有的模板解析和页面布局。

2. 第二层滤镜（单页骨架）：Page Layouts & Charts（模板继承）
   - 角色：提供预制承重墙模块（物理克隆模板的坐标骨架与占位符）。
   - 属性：单页非必有。如果存在则继承骨架，如果缺失则进入 Free Design（自由设计模式）。
   - 下游关系：为具体的页面填充提供坐标约束。

3. 第三层滤镜（单页填肉）：Page Rhythm（页面节奏）
   - 角色：决定单页的软装密度。dense（密集）允许画网格卡片；breathing（呼吸感）强制大留白，禁止多卡片网格。
   - 属性：单页必有（每页不同）。
   - 下游关系：结合前两层的约束，最终输出完整的 SVG 页面。

请将图表设计为清晰的分层管线（Pipeline），明确体现“宏观基调 -> 单页骨架 -> 单页细节”的降维执行过程。
